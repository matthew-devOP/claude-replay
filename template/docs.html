claude-replay Documentation

A local web dashboard for browsing, replaying, and exporting Claude Code, Cursor, and Codex session transcripts. Runs entirely on your machine — no data leaves your computer.

1. Getting Started

Installation

Install globally with npm to get the claude-replay command available everywhere:

npm install -g claude-replay

Or run without installing using npx:

npx claude-replay

Or run with Docker (no Node.js required):

docker compose up

Starting the App

Launch the web dashboard from any directory:

claude-replay

The app scans your home directory for Claude Code, Cursor, and Codex session files and serves the dashboard at http://localhost:7331.

Options

  • --port <n> Listen on a custom port (default: 7331)
  • --host <addr> Bind to a specific address (default: 127.0.0.1). Use 0.0.0.0 to listen on all interfaces (LAN access).

Example with custom port:

claude-replay --port 8080
Note: The app binds to 127.0.0.1 by default, so it is only reachable from your local machine. Your session files are never sent anywhere.

2. Project Dashboard (/)

The dashboard is the main landing page. It automatically discovers all Claude Code, Cursor, and Codex projects on your machine and groups their sessions by project directory.

Sidebar

The left sidebar lists all discovered projects. Each card shows:

  • Project name — derived from the directory name
  • Git branch — current branch indicator when a git repo is detected
  • Session count — total number of transcript files found
  • Last activity — date/time of the most recent session

Click a project card to open its detail view in the main panel.

Project Detail View

The detail view shows tabs for the selected project:

  • CLAUDE.md — displays the project's CLAUDE.md memory file if present
  • MEMORY.md — displays the project's MEMORY.md file if present
  • Sessions — sortable table of all sessions (see below)
  • Stats — aggregated tool usage statistics across all sessions
  • Plans — plan-mode entries extracted from sessions
  • Git — repository information

Sessions Tab

Displays all sessions for the project in a sortable table. Columns include:

  • Date — session creation time
  • Turns — number of conversation turns
  • Size — file size of the JSONL transcript
  • Actions — Replay, Transcript, Edit, MD Export buttons

Click any column header to sort. Click again to reverse the sort order.

Stats Tab

Aggregated statistics across all sessions in the project:

  • Tool usage bars — horizontal bars showing relative tool invocation frequency
  • Bash commands — most frequently run shell commands
  • Files read/edited — most frequently accessed files
  • Agents with prompts — subagent invocations and their system prompts
  • User input/output tokens — token usage across sessions
  • Teams — when multiple model types were used

Plans Tab

Lists plan-mode entries extracted from sessions. Plan mode is when Claude Code enters a structured planning phase before executing tasks. Each entry shows the plan text and the session it came from.

Git Tab

Shows git repository information for the project:

  • Current branch and upstream tracking status
  • Total commit count
  • Remote URL(s)
  • All local branches
  • Recent git history as a visual graph

Quick Actions

From the project detail header you can:

  • Open in Finder — reveal the project directory in macOS Finder
  • Open in Terminal — open a new terminal window at the project path

3. Session Actions

Each session row in the Sessions tab has four action buttons:

Replay

Opens the session in the Replay Viewer — an animated player that walks through the conversation turn by turn, simulating the original pace of the session. Useful for presenting or reviewing work.

Transcript

Opens a full-page conversation viewer. Features include:

  • Search — type to filter; use Enter to jump to the next match, Shift+Enter for the previous match
  • Content filters — toggle visibility of User, Assistant, Tools, and Thinking message types
  • Match count — shows how many results were found and which one is active

Edit

Opens the session in the Session Editor for customization before exporting. See Session Editor for details.

MD Export

Directly downloads the session as a Markdown (.md) file without opening any viewer. The Markdown output includes all turns formatted as readable conversation text.

4. Replay Viewer (/replay)

The Replay Viewer renders a session as an animated playback. It embeds a standalone player that streams conversation content turn by turn.

Player Controls

  • Play/Pause — start or pause the animation
  • Step forward/back — advance or rewind one turn at a time
  • Progress bar — click anywhere on the bar to jump to that position in the session

Speed Control

Adjust playback speed with the speed dropdown: 0.5x, 1x, 2x, 5x, 10x, 20x. Speed is applied to the inter-turn timing, so it compresses or expands the pauses between turns.

Keyboard Shortcuts

Export from Player

  • Markdown download — export the full conversation as a .md file directly from the player controls
  • PDF via print — use your browser's print dialog (Cmd+P / Ctrl+P) to save the current player state as a PDF

Theme Switching

The theme dropdown in the player header switches the visual theme in real-time. The selection is saved to localStorage and persists across all pages.

5. Session Editor (/editor)

The Session Editor is a three-panel workspace for customizing a session before exporting a replay.

Three-Panel Layout

  • Left — Turn Sidebar: Lists all turns in the session. Click to select and edit a turn.
  • Center — Turn Editor: Edit the selected turn's content, toggle inclusion, set bookmarks.
  • Right — Live Preview: A live-rendered replay preview that updates as you make changes.

Turn Editing

  • Include/Exclude — toggle individual turns on or off. Excluded turns are skipped during replay but remain in the file.
  • Edit user text — modify the text of user messages for cleaner replays.
  • Bookmarks — mark turns as bookmarks to create named chapters in the replay.

Export Options Panel

The export panel (right side) controls how the final replay HTML is generated:

  • Export Theme — the color theme baked into the exported file. This is independent of the UI Theme you use while editing.
  • Speed — default playback speed for the exported replay.
  • Thinking blocks — include or exclude extended thinking content.
  • Tool calls — include or exclude tool use and tool result turns.
  • Redaction — automatically redact detected secrets (API keys, tokens, passwords) from the export.
Export Theme vs UI Theme: The UI Theme controls the editor's own appearance and is saved via localStorage. The Export Theme is embedded into the generated replay HTML and travels with the file.

Exporting

Click Export HTML to download a fully self-contained replay file. The file includes all CSS, JavaScript, and conversation data — it requires no internet connection and can be shared as a single attachment.

6. CLI Usage

In addition to the web dashboard, claude-replay provides a command-line interface for generating replay files directly.

Generate HTML Replay

# From a session ID
claude-replay <session-id> -o replay.html

# From a JSONL file path
claude-replay /path/to/session.jsonl -o replay.html

# Multiple sessions concatenated
claude-replay id1 id2 id3 -o combined.html

Open in Editor

claude-replay editor /path/to/session.jsonl

Extract Transcript

# Extract as Markdown
claude-replay extract /path/to/session.jsonl

# Extract to file
claude-replay extract /path/to/session.jsonl -o transcript.md

Key Flags

  • --turns <n> Limit output to the first n turns
  • --speed <n> Default playback speed multiplier (e.g. 2, 5)
  • --theme <name> Embed a specific theme: tokyo-night, monokai, solarized-dark, github-light, dracula, bubbles
  • --redact Automatically redact detected secrets (API keys, tokens, passwords)
  • --no-thinking Strip extended thinking blocks from the output
  • --no-tools Strip tool use and tool result turns from the output
  • --timing <mode> Timing mode: auto (default), real (preserve original timing), paced (uniform pacing)
  • --custom-theme <path> Path to a custom theme JSON file
  • --list-themes Print all available built-in theme names and exit
  • --port <n> Port for the dashboard/editor server (default: 7331)

Timing Modes

ModeDescription
autoInfers timing from session metadata when available, otherwise uses paced timing
realPreserves the actual elapsed time between turns from the original session
pacedUses a uniform fixed delay between all turns regardless of original timing

8. Docker

A docker-compose.yml is included in the repository for running the app in a container without installing Node.js locally.

Starting

docker compose up

The app will be available at http://localhost:7331. Your home directory is mounted read-only into the container so sessions are discovered just as they would be natively.

Live Development

The docker-compose.yml mounts the source directory into the container, so changes to template files and source files are reflected without rebuilding the image:

# Restart the server process after source changes
docker compose restart

Path Resolution

Inside the container, session paths are resolved relative to the container's filesystem. The docker-compose.yml mounts $HOME at the same path inside the container so that session IDs and file paths match what Claude Code writes natively.

Tip: If you add new session files while the container is running, they will appear in the dashboard on the next project refresh without restarting.

9. Privacy & Security

Local Only

The server binds to 127.0.0.1 by default. No requests are made to any external service, and no data is transmitted over the network unless you explicitly use --host 0.0.0.0 to expose it on your LAN.

Read-Only Access

Your JSONL session files are never modified. The app reads them to render the dashboard and generate exports, but all edits in the Session Editor are held in memory and only written to disk when you explicitly export.

Secret Auto-Redaction

When the --redact flag is used (or the Redaction toggle is enabled in the editor), the following patterns are automatically detected and replaced with [REDACTED] before export:

  • API keys matching common provider patterns (OpenAI, Anthropic, AWS, etc.)
  • Bearer tokens and Authorization headers
  • Passwords and secrets in environment variable assignments
  • Private keys in PEM format
Review before sharing: Automatic redaction is best-effort. Always review exported files before sharing them externally, especially if your sessions involve credentials, internal URLs, or proprietary code.
